## 发现的问题

**根本原因：** 测试使用了 `@pytest.mark.asyncio` 装饰器，但测试运行器可能未正确配置异步测试。

**失败测试（7个）：**
- `test_acquire_consumes_token`
- `test_acquire_many_returns_zero_wait_initially`
- `test_acquire_beyond_rate_requires_wait`
- `test_tokens_refill_over_time`
- `test_context_manager`
- `test_context_manager_waits_when_needed`

**注意：** 不使用 `acquire()` 的测试通过：
- `test_initial_tokens` ✅
- `test_different_rates` ✅
- `test_max_tokens_capped` ✅

## 建议

### P0 - 严重（测试损坏）

1. **正确配置 pytest-asyncio** - 在 `pyproject.toml` 中添加：
   ```toml
   [tool.pytest.ini_options]
   asyncio_mode = "auto"
   ```

2. **修复异步测试模式** - `test_rate_limiter.py` 和 `test_api_key_rotator.py` 测试需要正确的异步处理

### P1 - 高优先级（模拟问题）

3. **改进 HTTP 模拟** - 使用 `respx` 或 `aioresponses` 进行异步 HTTP 模拟
4. **模拟 SEC Edgar HTTP 调用** - 所有 HTTP 依赖测试应使用正确的模拟

### P2 - 中优先级（集成）

5. **为 DataServer 添加测试夹具** - 为 API 测试创建模拟 DataServer
6. **为模型添加测试夹具** - 为预测器测试创建模拟模型文件
7. **优化测试执行** - 使用 `@pytest.mark.slow` 标记慢速测试并单独运行
